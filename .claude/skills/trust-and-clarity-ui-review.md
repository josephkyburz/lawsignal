# Purpose

LawSignal should communicate quickly that it helps users evaluate and compare law firms. The interface should feel credible, readable, and low-friction before it feels clever.

# Use When

- Reviewing homepage, onboarding, filters, results, or About copy
- Evaluating design changes that may affect comprehension
- Tightening user-facing language or calls to action

# Review Lens

## Five-second test

Within a few seconds, can a first-time visitor tell:

- this is a law firm research and ranking tool
- it helps match firms to user priorities
- the next action is clear

If not, the change needs revision.

## Non-negotiables

- Clarity over novelty
- Credibility over hype
- Accessibility is part of done, not a polish pass
- Keep user-facing claims specific and supportable
- Prefer direct labels over clever CTA copy

## UI checks

- Headings are direct and descriptive
- Important choices are easy to scan on mobile and desktop
- Focus states remain visible
- Motion is restrained and not required for understanding
- Contrast is strong enough for controls, chips, and score labels
- Layout does not create unnecessary cognitive load

## Copy checks

- Prefer plain language over marketing phrasing
- Do not overclaim precision or authority
- Avoid vague labels like "Learn more" where a direct label would help
- Keep trust signals compact and concrete

# Guardrails

- Do not invent legal, compensation, market, or methodology claims
- Do not add visual flourish that competes with comprehension
- Do not hide critical scoring or filter behavior behind ambiguous UI
